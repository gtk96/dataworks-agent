# DataWorks Agent UI 简洁化设计（New API 风格）

日期：2026-07-22  
仓库：`E:\dataworks_agent`  
分支：`chat-first-ui`（与 `main` 同提交 `0b63c567c`，UI 工作均为未提交修改）  
状态：待用户审阅

## 1. 目标与范围

### 1.1 目标

把当前“营销型 AI 首页”改造成 **New API 式后台控制台**：

- 简洁、白底、紧凑、统一控件
- 对话优先，但管理入口齐全且不抢主视觉
- 操作路径短：登录 → 选连接/项目 → 提问/SQL → 看结构化结果
- 体验流畅：少跳转、即时反馈、错误可恢复

### 1.2 范围内

- 应用壳层（侧栏 + 顶栏 + 内容区）
- 对话首页 / 会话页 / SQL 与 Skill 结果展示
- 连接与项目上下文选择
- MCP / 连接 / 任务 / 技能 / 知识 / 审计 / 设置入口整合
- 视觉与交互规范（间距、颜色、状态、反馈）
- 清理当前未完成的 hero 装饰与重复样式实现

### 1.3 范围外

- 后端协议大改
- 新增大模型能力
- 重写 OpenCode 全量 IDE 能力
- 独立移动端 App

### 1.4 非目标（明确不做）

- 大标题英雄区、渐变光晕、胶囊堆砌
- 全屏“AI 欢迎页”感
- 把所有管理功能塞进聊天输入周围
- 只做换肤不改信息架构

## 3. 设计决策

### 3.1 布局：聚焦对话 + New API 壳层

采用：

```
┌──────────────┬──────────────────────────────────────────────┐
│ 侧栏导航      │ 顶栏：面包屑 + 页面动作                        │
│              ├──────────────────────────────────────────────┤
│ 对话          │ 白底内容区                                    │
│ 数据连接      │  - 状态指标卡                                  │
│ 数据探索      │  - 查询输入 / 会话流                            │
│ 任务中心      │  - 结果卡片 / 表格                              │
│ MCP           │                                              │
│ 技能          │                                              │
│ 知识库        │                                              │
│ 审计          │                                              │
│ 设置          │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 3.2 侧栏信息架构

固定入口（从上到下）：

1. **对话**（默认首页）
2. **数据连接**
3. **数据探索**
4. **任务中心**
5. **MCP**
6. **技能**
7. **知识库**
8. **审计**
9. **设置**

规则：

- 侧栏固定，路由切换不换信息架构
- 当前页：浅蓝底 + 主色文字
- 移动端：侧栏收成抽屉
- 次要能力不进聊天输入区周围

### 3.3 视觉语言（对齐 New API）

- 背景：浅灰内容底 `#f7f8fa`，卡片/侧栏白底 `#ffffff`
- 边框：`#e5e7eb` / `#e8eaef`
- 主色：仅用于激活态、主按钮、关键链接（`#2563eb`）
- 成功/警告/危险：绿 / 橙 / 红，只用于状态标签
- 圆角：控件 6px，卡片 8–10px，标签 999px
- 字体：系统无衬线；辅助信息可用等宽（SQL、ID、环境）
- 阴影：极轻，不用大光晕
- 密度：中等偏紧凑，首屏信息可读但不拥挤

### 3.4 实现路径

推荐：

**在现有 `chat-first-ui` 上重做壳层与对话表面**，复用控制面与会话能力，不做整仓换壳。

## 4. 关键设计

### 4.1 登录页

- 居中白卡片，最大宽约 360–400px
- 字段：用户名、密码、主按钮「登录」
- 错误：卡片内红字提示（含限流）
- 登录成功默认进入 **对话**
- 文案使用“用户名”，不是邮箱（与当前本地 admin 一致）

### 4.2 对话首页（空状态）

结构：

1. 顶栏：`控制台 / 对话` + `历史记录` + `新建对话`
2. 四张状态卡：
   - 当前连接
   - 当前项目
   - 今日查询（可先占位，无数据时显示 `--`）
   - 写权限（开/关）
3. 「开始查询」白卡片：
   - 连接选择
   - 项目选择
   - 模式（只读 / 可写）
   - 多行输入框
   - 快捷 chips + 发送按钮
4. 「常用操作」2×2：
   - 搜索业务表
   - 查看失败任务
   - 订单成本诊断
   - 连通性检查（`SELECT 1`）

行为：

- 未选连接/项目：发送禁用，输入区旁即时提示
- Enter 发送，Shift+Enter 换行
- chips / 常用操作：一键填入输入框并聚焦，不自动连发（避免误操作）
- 若后端已支持直接执行的固定动作，可在按钮上提供“填入并发送”二级确认，默认仍是填入

### 4.3 会话页（进行中）

- 左侧导航不变
- 主区：消息流 + 底部固定输入条
- 用户消息：右对齐浅灰气泡
- 助手消息：左对齐白卡片
- 流式状态：安静步骤列表（完成/进行中），不做重动画
- 顶栏显示会话标题（可用首条用户问题截断）

### 4.4 结果展示

#### SQL 结果卡（`dw_run_sql`）

- 标题：SQL 查询
- 副标题：行数 / 耗时 / 项目
- 可折叠 SQL 原文
- 表格展示结果（默认前 50 行，可展开/分页）
- 操作：复制 SQL、复制结果、在数据探索打开（若路由可用）
- 失败：错误卡 + 建议（权限 / 连接 / 语法）

#### Skill 查数结果（物流 / 采购异常）

- 摘要区：问题结论 1–3 句
- 数据表：关键字段
- 订单标签
- follow-up 按钮（如「查费率来源」「根因诊断」）
- 进度步骤可折叠，默认收起已完成步骤

#### 通用错误

- 红边提示卡
- 文案可读，不暴露堆栈
- 提供动作：重试 / 去连接页 / 检查 Cookie/登录态

### 4.5 数据连接页

- 列表 + 状态标签 + 写权限开关 + 管理动作
- 与对话页连接选择共用同一数据源
- 空状态引导：新增连接

### 4.6 MCP 页

- 列出 MCP 服务/工具状态
- 可见：启用、健康、最近错误
- 从侧栏直达，不塞进对话输入区

### 4.7 其他管理页

数据探索、任务中心、技能、知识库、审计、设置：

- 统一使用同一壳层
- 页面内采用「页头说明 + 工具条 + 白卡片内容」
- 保持现有业务能力，先统一壳与视觉，不借机重做业务逻辑

## 5. 交互与状态

### 5.1 全局状态

| 状态 | 表现 |
|---|---|
| 加载中 | 内容区骨架或局部 spinner，不整页白屏 |
| 空数据 | 简洁空状态 + 一个主操作 |
| 部分失败 | 警告条 + 重试 |
| 限流 | 明确“稍后再试” |
| 未登录 | 跳转 `/login`，保留 `returnTo` |
| 无权限 | 禁止页 + 返回安全入口 |

### 5.2 对话状态

| 状态 | 表现 |
|---|---|
| 可发送 | 主按钮高亮 |
| 上下文缺失 | 禁用发送 + 提示补齐连接/项目 |
| 生成中 | 输入禁用或显示停止（若后端支持）；步骤流更新 |
| 成功 | 结果卡插入消息流，自动滚到底 |
| 失败 | 错误卡，保留用户输入便于重发 |
| 历史预览 | 顶部 info 条：只读，不可续聊 |

### 5.3 快捷键

- `Enter` 发送
- `Shift+Enter` 换行
- `Ctrl/Cmd+K` 命令面板（沿用现有能力，若已有）
- 侧栏支持键盘聚焦与回车激活（基础可访问性）

## 7. 视觉与文案规范

### 7.1 文案原则

- 短、明确、可执行
- 中文优先，专业词可保留英文（SQL、MCP）
- 错误给下一步，不只报失败

示例：

- 好：`请先选择连接和项目，再发送查询`
- 差：`参数错误`
- 好：`Cookie 失效，请到数据连接页重新校验`
- 差：`request failed`

### 7.2 间距

- 页面内容左右 16–20px
- 卡片内 12–16px
- 卡片间距 12px
- 表单控件高度约 32–36px

## 8. 错误处理与边界

1. 未登录访问受保护路由 → `/login?returnTo=...`
2. 连接列表失败 → 指标卡显示失败 + 重试，不阻断浏览壳层
3. 项目为空 → 项目选择显示空态，发送禁用
4. SQL 超长结果 → 截断展示 + 展开/分页，避免页面卡死
5. 历史只读预览 → 禁止继续在该上下文发送
6. 写权限关闭时 → 模式固定只读，尝试写操作前提示
7. MCP 离线 → MCP 页与状态区可见，不静默失败

## 11. 成功标准

用户体感上应满足：

1. **看起来像 New API 后台，不像 AI 营销站**
2. **3 步内能完成一次有效查询**（选连接/项目 → 输入 → 看结果）
3. **结果可读**：表格/卡片优先，不是长文本糊屏
4. **管理入口找得到**：MCP 等不缺，且不干扰主操作
5. **稳定流畅**：加载、失败、重试路径清楚

## 12. 风险与缓解

| 风险 | 缓解 |
|---|---|
| OpenCode 原布局与 New API 壳冲突 | 仅在 DataWorks 产品表面套壳，不强改全部 IDE 视图 |
| 现有未提交改动质量不稳 | 先收敛重复实现，再加新壳；改一段验一段 |
| SQL 结果过大 | 默认截断 + 分页/展开 |
| 入口过多导致侧栏拥挤 | 文案短、分组清晰；先全量展示，后续再折叠次要项 |
